LIGHT

In this project I analyzed the successes and failures of my refrigerator water dispenser interface.

I analyzed the interface by creating two personas that represent different desires of people I interviewed.

I created a storyboard for one of these personas and walked through how they might interact and respond to the interface.

I concluded that this interface is a bad interface because it was very rare that someone succesfully interacted with it on their first attempt.

PERSONAS

I analyzed and re-designed a website of a self-storage service in Kentucky.

Franklin's self storage was a not-responsive website that was quite difficult to navigate dispite having a very simple functionality

In redesigning this website, I wanted to keep it simple while maintaining the functionality.

I used responsive design principles so that the website will work with any size screen.

REDESIGN

This was a group project where we selected a startup named Ozone and prototyped a posibility of their software using Figma.

Ozone's goal is to create a collaborative video editing software similar to Figma (but for videos)

To create the design of this software, we combined ideas from collaborative tools (such as Google Drive and Figma) as well as standard video editing tools (such as iMovie)

We started with sketches and turned those sketches into initial figma prototypes. We refined this prototype and then went through the user testing stage.

We then got feedback from the users and refined the prototype to better suit our needs.

ITERATIVE DESIGN

The deployment project was intended to give us practice in implementing a website using html, css and javascript.

The goal was to create a shopping center where you can sort the items, you can filter the items and you can add the items to the cart.

Although it was a very simple project, it provided a great foundation for future javascript implementations.
